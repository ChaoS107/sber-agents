1. **Название проекта**

ИИ-ассистент по банковскому обслуживанию корпоративных клиентов  
AI Assistant for Corporate Banking Services

2. **Описание проекта**

Интеллектуальный ассистент, помогающий корпоративным клиентам решать задачи дистанционного банковского обслуживания посредством современных AI-технологий. Ассистент способен анализировать данные клиента, предоставлять аналитику по операциям, консультировать по продуктам, помогать с оформлением документов и интегрирован с мультиканальными интерфейсами (чат, голос, изображение).

3. **Целевая аудитория**

Корпоративные клиенты банка: представители финансовых и бухгалтерских служб, сотрудники, ответственные за взаимодействие с банком, специалисты по управлению счетами и казначейству компаний среднего и крупного бизнеса.

4. **Основные функции и возможности**

- Просмотр баланса по корпоративным счетам
- Аналитика по тратам и доходам (графики, отчёты)
- Консультации по банковским продуктам (кредиты, депозиты, гарантии и пр.)
- Оформление заявок (например, на выпуск корпоративных карт)
- Распознавание и обработка документов (загрузка сканов, извлечение ключевых данных)
- Поддержка по чату (в дальнейшем — голосовая поддержка)
- Персонализированные рекомендации (в дальнейшем)
- Интеграция с системами банка через API (используются демо-данные и тестовые сценарии)
- Мультимодальный интерфейс (текст, голос, изображения)

5. **Технологии и подходы из курса**

- Работа с LLM через API (OpenAI-совместимые интерфейсы):
  Используется для организации диалогов, генерации ответов и сценариев взаимодействия с клиентом. Ассистент будет строить коммуникацию, опираясь на мощные языковые модели.

- Structured Output — структурированный вывод:
  Для выполнения банковских операций, выдачи аналитических справок и формирования документов AI будет возвращать структурированные данные (JSON- или tabular-ответы), которые легко интегрировать в интерфейс или передать в другие системы.

- Мультимодальность (голос, изображения):
  Ассистент сможет не только поддерживать диалог в текстовом, но и в голосовом режиме (TTS/STT), а также анализировать документы и изображения (например, распознавание и обработка сканов документов).

- RAG (Retrieval-Augmented Generation):
  Для консультаций и справок ассистент будет обращаться к базе знаний (например, FAQ, внутренние инструкции или справочные материалы банка), обеспечивая актуальные и точные ответы даже по редким или изменяющимся вопросам.

- Агентные паттерны и сложные workflow с LangGraph:
  Оркестрация сложных сценариев – например, консультация, затем сбор необходимых данных, далее заполнение формы, подтверждение и отправка заявки – реализуется с помощью agent-паттернов и графов сценариев.

- MCP (Model Context Protocol) — интеграция с внешними инструментами:
  Для расширения возможностей ассистента обеспечивается связь с внешними API (например, для проверки баланса, статусов заявок, получения информации по продуктам).

6. **План развития проекта по модулям курса**

- Модуль 1: Работа с LLM через API
  - Создание базового чат-ассистента для типовых диалогов с корпоративными клиентами (информация о продуктах, ответы на частые вопросы).

- Модуль 2: Structured Output
  - Реализация структурированных ответов: вывод балансов и аналитики в виде таблиц, автоматическая разметка и сведение данных по запросу клиента.

- Модуль 3: Мультимодальность (голос, изображения)
  - Добавление поддержки голоса (озвучивание и распознавание реплик), а также возможности загружать и распознавать сканы документов для обработки банковских заявок.

- Модуль 4: RAG (Retrieval-Augmented Generation)
  - Интеграция с базой знаний банка для поиска нормативной/справочной информации (регламенты, инструкции, продуктовые описания) и ответа на нетривиальные вопросы.

- Модуль 5: Агентные паттерны и сложные workflow с LangGraph
  - Построение сложных сценариев: например, согласование заявки, пошаговое заполнение и отправка документов с контролем состояний и возвратом на этапы.

- Модуль 6: MCP (интеграция с внешними инструментами)
  - Связь чат-ассистента с внешними системами для имитации реальных банковских процессов: получение баланса по API, статус заявок, работа с демо-данными, автоматизированный документооборот.
